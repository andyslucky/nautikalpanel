Nautikal Panel is a Kubernetes-native game server management platform designed with simplicity at its core. It transforms the complex process of provisioning and managing game servers into a streamlined, intuitive experience that gets you as close to "1-click" deployments as possible.

## The Problem Nautikal Panel Solves

Running game servers on Kubernetes offers incredible advantages—scalability, resilience, and cloud-native tooling—but it comes with significant overhead. You typically need to write and maintain Kubernetes YAML manifests, configure persistent volumes and storage classes, set up networking with services and ingress, manage secrets and credentials securely, handle pod lifecycle and resource allocation, and monitor logs and troubleshoot issues. Nautikal Panel abstracts all of this complexity away, letting you focus on what matters: your games and your community.

## How Nautikal Panel Works

Nautikal Panel sits between you and your Kubernetes cluster, handling all the orchestration details automatically. Here's what happens when you create a game server:

1. **Select a template** — Choose from pre-configured game server templates (Minecraft, Valheim, ARK, and more) or create your own
2. **Configure settings** — Adjust resources, storage, and network options through a simple web interface
3. **Deploy** — Click "Create" and Nautikal Panel automatically:
   - Creates Kubernetes resources (pods, services, persistent volumes, secrets)
   - Configures storage and networking
   - Generates secure credentials for file access
   - Sets up log streaming and monitoring
4. **Manage** — Start, stop, monitor, and access your servers with a single click

That's it. No YAML, no kubectl commands, no infrastructure complexity.

## The Simplicity Philosophy

Nautikal Panel is built around these principles:

### 1-Click Operations

Every common action is a single click away. Start a server to spin up a fully configured game server pod, stop it to cleanly shut down while preserving data, access files by starting SFTP mode with auto-generated credentials, view logs through real-time streaming, or delete a server to remove all associated resources.

### Template-Based Deployment

Templates encapsulate all the knowledge needed to run a specific game, including container images and versions, resource requirements and limits, network ports and protocols, environment variables and game-specific settings, and storage paths and sizing. Once a template exists, deploying a new server is essentially one click away.

### Automatic Resource Management

Nautikal Panel handles all Kubernetes lifecycle management, including automatic persistent volume provisioning and attachment, dynamic service creation with LoadBalancer support, secure credential generation and secret management, pod status monitoring, and clean resource deletion and cleanup.

## Key Features

### Template Repository System

Organize and manage your game server templates through repositories. Use local repositories to store templates in a directory, or GitHub repositories to share and download templates directly. Create once, deploy anywhere—templates define everything about a game server, making it trivial to spin up new instances.

### One-Click Server Lifecycle

From creation to deletion, every step is designed for speed and simplicity. Create by selecting a template and naming your server, start to deploy the game server pod with all dependencies including the SFTP sidecar container, stop to shut down the pod while preserving data and configuration, enable SFTP mode to start a lightweight pod for file management, or delete to remove all Kubernetes resources and data.

### Built-in File Management

Every server includes secure SFTP access with automatic credential generation, SFTP-only mode for lower resource usage when managing files before launch, direct access to persistent volumes, and compatibility with any standard SFTP client.

### Real-Time Monitoring

Stay informed about your servers without leaving the browser. The dashboard updates automatically as pod status changes, WebSocket-based log streaming shows server output in real-time, WebSocket watchers push pod changes instantly to the UI, and visual indicators show running, pending, or offline states.

### Kubernetes Native

Nautikal Panel leverages Kubernetes' full capabilities, including scalability to run as many servers as your cluster supports, resilience with automatic pod restart on failures, resource isolation through CPU and memory limits, portability across any Kubernetes cluster, and integration with existing Kubernetes tools for monitoring, logging, and networking.

## Architecture

Nautikal Panel consists of three core components:

### Backend (Rust)

The backend provides the REST API and handles all Kubernetes orchestration, managing pods, services, PVCs, and secrets through the Kubernetes client, rendering Kubernetes manifests from templates via the template engine, storing game server configurations in SurrealDB, and providing real-time log streaming and status updates through WebSocket servers.

### Frontend

A lightweight web interface built with Alpine.js provides a dashboard with server overview and status indicators, server management for creating, editing, starting, stopping, and deleting servers, a real-time log streaming modal, and user preferences and configuration settings.

### Kubernetes

The execution backend where game servers actually run, featuring game server containers with resource limits as pods, LoadBalancer services for external access, persistent volume claims for game data storage, and secure storage for SFTP credentials in secrets.

## Use Cases

Nautikal Panel is designed for anyone who needs to run game servers though its focus is on gamers who dont want to spend money on
game server hosting. It is designed to work with flexibility in mind, and any gaming community can create templates for game servers
they host or think others would like to host.
