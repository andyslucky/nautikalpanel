Creating a game server in Nautikal Panel is straightforward thanks to the template system. This guide covers all the options available.

## Opening the Create Dialog

1. Click **+ Add Server** on the dashboard
2. The Create Server modal opens with multiple tabs

## General Tab

Basic server identification and settings.

### Server Name

- A friendly name to identify this server
- Shown in the dashboard and logs
- Can be changed later via Edit

### Template Selection

Select from available templates in the dropdown:
- Templates define default configuration for a game type
- Template selection populates all other fields
- You can modify any field after selection

### Game Version

- Optional field for tracking
- Not used by Nautikal Panel directly
- Useful for documentation

### Max Players

- Optional player limit
- Some games use this for configuration
- Informational only for others

### Game Type

- Read-only (derived from template)
- Used for Kubernetes resource naming

### Icon URL

- Optional URL for server icon
- Displayed in dashboard
- Can be any accessible image URL

### Description

- Optional server description
- For your reference only

## Pod Config Tab

Container and resource configuration.

### Container Image

The Docker image to run. Examples:
- `itzg/minecraft-server` for Minecraft
- `lloesche/valheim-server` for Valheim
- Any valid Docker image reference

### CPU Resources

Dual sliders for CPU allocation:

- **Request**: Guaranteed CPU (millicores)
- **Limit**: Maximum CPU allowed

Example values:
- `250m` = 0.25 cores
- `1000m` = 1 core
- `2000m` = 2 cores

### Memory Resources

Dual sliders for memory:

- **Request**: Guaranteed memory
- **Limit**: Maximum memory allowed

Example values:
- `256Mi` = 256 megabytes
- `1Gi` = 1 gigabyte
- `4Gi` = 4 gigabytes

### Command

Custom command to run in the container:
- Enter comma-separated values
- Example: `java, -Xms1G, -Xmx4G, -jar, server.jar`
- Leave empty for image default

### Environment Variables

Key-value pairs passed to the container:

| Key | Value |
|-----|-------|
| `EULA` | `TRUE` |
| `VERSION` | `1.20.4` |
| `DIFFICULTY` | `normal` |

Click **Add Env Var +** to add new entries.

## Storage Config Tab

Persistent storage configuration.

### Storage Size

Amount of disk space allocated:
- Number + unit (Mi or Gi)
- Cannot be changed after creation
- Choose based on game requirements

### Size Unit

- **Mi**: Megabytes
- **Gi**: Gigabytes

### Container Path

Where storage is mounted in the container:
- Common paths: `/data`, `/server`, `/world`
- Must match what the game expects
- Template provides sensible default

### Storage Class

Optional Kubernetes storage class:
- Leave empty for cluster default
- Specify for special storage needs (SSD, NFS, etc.)

## Service Config Tab

Network and service exposure.

### Service Type

| Type | Use Case |
|------|----------|
| **LoadBalancer** | Cloud environments with external LB |
| **NodePort** | Bare metal, exposes on node IP |
| **ClusterIP** | Internal only, requires port-forward |

### IP Address

Optional specific IP for LoadBalancer:
- Leave empty for automatic assignment
- Specify for static IP requirements

### Ports

Ports to expose:

| Field | Description |
|-------|-------------|
| **Port** | External port number |
| **Protocol** | TCP or UDP |

Click **Add Port +** for multiple ports.

## Misc Tab

Advanced template overrides.

:::warning
These options are for advanced users. Incorrect values can cause failures.
:::

### Init Template

Path to Jinja template for initial resources:
- Creates PVC and Service
- Default: `default/init.yaml.jinja`
- Override for custom resource creation

### Pod Template

Path to Jinja template for the pod:
- Creates the game server pod
- Default: `default/pod_template.yaml.jinja`
- Override for custom pod configuration

## Creating the Server

Click **Create** to:
1. Save the server configuration to the database
2. Create initial Kubernetes resources (PVC, Service)
3. Return to dashboard with server showing "Offline" status

The server is now ready to be started.

## Editing After Creation

Servers can only be edited when offline:
1. Stop the server if running
2. Click **Edit** on the server card
3. Modify available fields (not all are editable)
4. Click **Save**

### Editable Fields

- Server name
- Game version
- Max players
- Icon URL
- Description
- Pod configuration (image, resources, env, command)
- Pod template path

### Non-Editable Fields

These are set at creation and cannot be changed:
- Game type
- Storage configuration
- Service configuration
- Init template
